**src/lib/evaluation/orchestrator.ts:**

```typescript
import { type DimensionEvaluation, type ProposalEvaluation, type EvaluationDimension, DIMENSIONS } from "./schemas";
import { evaluateDimension } from "./agents";
import { DIMENSION_WEIGHTS } from "./constants";
import { pinEvaluationToIPFS, publishScoreOnChain } from "./storage";
```

**`computeAggregateScore(dimensions: DimensionEvaluation[]): number`**
1. Sum the weighted scores: `dimensions.reduce((sum, dim) => sum + dim.output.score * DIMENSION_WEIGHTS[dim.dimension], 0)`
2. If fewer than 4 dimensions, compute the total weight present: `dimensions.reduce((sum, dim) => sum + DIMENSION_WEIGHTS[dim.dimension], 0)`
3. Re-normalize: `weightedSum / totalWeight * 1.0` (so 3 of 4 dimensions still produces a 0-100 score)
4. Round to 1 decimal place: `Math.round(result * 10) / 10`

**`orchestrateEvaluation(proposalId: string, proposalText: string, onProgress: (event: EvaluationProgressEvent) => void): Promise<ProposalEvaluation>`**

Define the progress event type:
```typescript
export type EvaluationProgressEvent =
  | { type: "started"; proposalId: string; totalDimensions: number }
  | { type: "dimension_complete"; dimension: EvaluationDimension; output: EvaluationOutput; completedCount: number }
  | { type: "dimension_failed"; dimension: EvaluationDimension; error: string; completedCount: number }
  | { type: "aggregate_computed"; weightedScore: number; completedDimensions: number }
  | { type: "stored"; ipfsCid: string; txHash: string }
  | { type: "complete"; evaluation: ProposalEvaluation }
  | { type: "failed"; error: string };
```

Implementation:
1. Emit `{ type: "started", proposalId, totalDimensions: 4 }`
2. Create 4 dimension promises using `DIMENSIONS.map(...)`. Each promise: call `evaluateDimension(dim.key, proposalText)`, on success emit `dimension_complete` with incrementing `completedCount`, on failure emit `dimension_failed` and return null.
3. Use `Promise.allSettled` to await all 4. Filter out null (failed) results.
4. If zero successes, emit `{ type: "failed" }` and throw.
5. Compute aggregate: `const weightedScore = computeAggregateScore(successfulDimensions);`
6. Emit `{ type: "aggregate_computed", weightedScore, completedDimensions: successfulDimensions.length }`
7. Build ProposalEvaluation object with status "evaluated".
8. Pin to IPFS: `const ipfsCid = await pinEvaluationToIPFS(evaluation);`
9. Publish on-chain: `const txHash = await publishScoreOnChain(proposalId, weightedScore, ipfsCid);`
10. Emit `{ type: "stored", ipfsCid, txHash }`
11. Emit `{ type: "complete", evaluation }`
12. Return evaluation.

Use a mutable counter (`let completedCount = 0`) incremented atomically per dimension completion for the progress events.

CLAUDE.md compliance: viem's `privateKeyToAccount` and `writeContract` require `\`0x\${string}\`` typed arguments. Do NOT use `as` casts. Instead, define a type-safe helper using a type predicate:

```typescript
function isHexString(value: string): value is `0x${string}` {
  return /^0x[0-9a-fA-F]+$/.test(value);
}

function requireHexString(value: string, name: string): `0x${string}` {
  const prefixed = value.startsWith("0x") ? value : `0x${value}`;
  if (!isHexString(prefixed)) {
    throw new Error(`${name} is not a valid hex string: ${value}`);
  }
  return prefixed; // type narrowed by isHexString guard, no assertion needed
}
```

Use `requireHexString` wherever viem needs `\`0x\${string}\`` (private key, contract address).

**`POST` handler:**

1. Parse request body: `const body = await request.json()`. Validate with Zod: `z.object({ proposalId: z.string().min(1), proposalText: z.string().min(1) })`. Return 400 with `{ error: "..." }` on validation failure.

2. Create a `ReadableStream` for SSE:
```typescript
const stream = new ReadableStream({
  async start(controller) {
    const encoder = new TextEncoder();
    const sendEvent = (event: EvaluationProgressEvent) => {
      controller.enqueue(encoder.encode(`data: ${JSON.stringify(event)}\n\n`));
    };

    try {
      const evaluation = await orchestrateEvaluation(
        body.proposalId,
        body.proposalText,
        sendEvent,
      );
      // Final event already sent by orchestrator
      controller.close();
    } catch (error) {
      sendEvent({ type: "failed", error: error instanceof Error ? error.message : "Unknown error" });
      controller.close();
    }
  },
});
```

3. Return SSE response:
```typescript
return new Response(stream, {
  headers: {
    "Content-Type": "text/event-stream",
    "Cache-Control": "no-cache",
    "Connection": "keep-alive",
  },
});
```

**Rate limiting (persistent via Upstash Redis):** Before starting the orchestration:
1. Apply `evaluationTriggerLimiter` keyed by IP (10 evaluations/hour/IP). Return 429 with Retry-After header if exceeded.
2. Apply `globalEvaluationLimiter` keyed by "global" (10 concurrent globally). Return 429 if exceeded.
3. Check per-proposalId idempotency: if evaluation already exists for this proposalId and status is "complete", return the existing result. If status is "streaming", return 409 Conflict.

```typescript
// Per-IP rate limit
const ip = request.headers.get("x-forwarded-for") ?? request.headers.get("x-real-ip") ?? "unknown";
const { success, reset } = await evaluationTriggerLimiter.limit(ip);
if (!success) {
  return Response.json({ error: "RATE_LIMITED", retryAfter: Math.ceil((reset - Date.now()) / 1000) }, { status: 429 });
}
```
