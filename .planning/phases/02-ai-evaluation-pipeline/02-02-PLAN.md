Define the progress event type. Addresses review concern: Added `storage_failed` event type so the client knows when evaluation succeeded but IPFS/chain storage failed.

```typescript
export type EvaluationProgressEvent =
  | { type: "started"; proposalId: string; totalDimensions: number }
  | { type: "dimension_complete"; dimension: EvaluationDimension; output: EvaluationOutput; completedCount: number }
  | { type: "dimension_failed"; dimension: EvaluationDimension; error: string; completedCount: number }
  | { type: "aggregate_computed"; weightedScore: number; completedDimensions: number }
  | { type: "storage_failed"; step: "ipfs" | "chain"; error: string }
  | { type: "stored"; ipfsCid: string; txHash: string }
  | { type: "complete"; evaluation: ProposalEvaluation }
  | { type: "failed"; error: string };
```

**`computeAggregateScore(dimensions: DimensionEvaluation[]): number`**
1. Sum the weighted scores: `dimensions.reduce((sum, dim) => sum + dim.output.score * DIMENSION_WEIGHTS[dim.dimension], 0)`
2. Compute the total weight present: `dimensions.reduce((sum, dim) => sum + DIMENSION_WEIGHTS[dim.dimension], 0)`
3. Re-normalize: `weightedSum / totalWeight` (so 3 of 4 dimensions still produces a 0-100 score)
4. Round to 1 decimal place: `Math.round(result * 10) / 10`

**`orchestrateEvaluation(proposalId: string, proposalText: string, onProgress: (event: EvaluationProgressEvent) => void): Promise<ProposalEvaluation>`**

Implementation:
1. Emit `{ type: "started", proposalId, totalDimensions: 4 }`
2. Create a mutable counter: `let completedCount = 0;`
3. Create 4 dimension promises using `DIMENSION_KEYS.map(...)`. Each promise wraps `evaluateDimension(dim, proposalText)` in a try/catch:
   - On success: increment `completedCount`, emit `{ type: "dimension_complete", dimension: dim, output: result.output, completedCount }`, return result.
   - On failure: increment `completedCount`, emit `{ type: "dimension_failed", dimension: dim, error: err.message, completedCount }`, return null.
4. Use `Promise.allSettled` to await all 4. Extract non-null results as `successfulDimensions`.
5. If zero successes, emit `{ type: "failed", error: "All 4 judge agents failed" }` and throw an error.
6. Compute aggregate: `const weightedScore = computeAggregateScore(successfulDimensions);`
7. Emit `{ type: "aggregate_computed", weightedScore, completedDimensions: successfulDimensions.length }`
8. Build ProposalEvaluation object:
   ```typescript
   const evaluation: ProposalEvaluation = {
     proposalId,
     dimensions: successfulDimensions,
     aggregate: { weightedScore, completedDimensions: successfulDimensions.length, computedAt: Date.now() },
     status: "evaluated",
   };
   ```
9. Storage with error handling (addresses review concern: no retry for IPFS/chain failures):
   ```typescript
   let ipfsCid = "storage-failed";
   let txHash = "storage-failed";
   try {
     ipfsCid = await pinEvaluationToIPFS(evaluation);
   } catch (storageErr) {
     onProgress({ type: "storage_failed", step: "ipfs", error: storageErr instanceof Error ? storageErr.message : "IPFS pin failed" });
   }
   try {
     txHash = await publishScoreOnChain(proposalId, weightedScore, ipfsCid);
   } catch (storageErr) {
     onProgress({ type: "storage_failed", step: "chain", error: storageErr instanceof Error ? storageErr.message : "Chain publish failed" });
   }
   ```
10. Emit `{ type: "stored", ipfsCid, txHash }`
11. Emit `{ type: "complete", evaluation }`
12. Return evaluation.

**`pinEvaluationToIPFS(evaluation: ProposalEvaluation): Promise<string>`**
1. Read `process.env.PINATA_JWT`. If not set, `console.warn("PINATA_JWT not set -- IPFS pinning disabled")` and return `"ipfs-disabled"`.
2. Fetch `https://api.pinata.cloud/pinning/pinJSONToIPFS` with:
   - Method: POST
   - Headers: `Authorization: Bearer ${process.env.PINATA_JWT}`, `Content-Type: application/json`
   - Body: `JSON.stringify({ pinataContent: evaluation, pinataMetadata: { name: \`evaluation-\${evaluation.proposalId}\` } })`
3. If response not ok, throw Error with status and statusText.
4. Parse response JSON and return `data.IpfsHash` (the CID string).

**`publishScoreOnChain(proposalId: string, score: number, ipfsCid: string): Promise<string>`**
1. Read env vars: `PRIVATE_KEY`, `NEXT_PUBLIC_REPUTATION_REGISTRY_ADDRESS`, `NEXT_PUBLIC_RPC_URL`.
2. If any are missing, `console.warn("Chain env vars not set -- on-chain publishing disabled")` and return `"chain-disabled"`.
3. Create account: `const account = privateKeyToAccount(\`0x\${process.env.PRIVATE_KEY}\` as \`0x\${string}\`);`
4. Create wallet client with `baseSepolia` chain and `http(process.env.NEXT_PUBLIC_RPC_URL)` transport.
5. Call `writeContract` on the ReputationRegistry:
   - address: `process.env.NEXT_PUBLIC_REPUTATION_REGISTRY_ADDRESS as \`0x\${string}\``
   - abi: REPUTATION_REGISTRY_ABI (defined below)
   - functionName: "giveFeedback"
   - args:
     - tokenId: `BigInt(proposalId)` -- NOTE: Phase 1 must ensure proposalId is a numeric string matching the on-chain token ID. If Phase 1 uses non-numeric proposal IDs, this line must be adapted. The coupling is documented here for Phase 1 awareness.
     - metricKey: `METRIC_KEY_EVALUATION_SCORE`
     - value: `BigInt(Math.round(score * 100))` (store as basis points for precision)
     - comment: `ipfs://${ipfsCid}`
6. Return the transaction hash string.

Addresses review concerns:

- `z.string().max(10000)` bounds proposalText input size

```typescript
import { z } from "zod";
import { orchestrateEvaluation, type EvaluationProgressEvent } from "@/lib/evaluation/orchestrator";

export const runtime = "nodejs";

// Addresses review concern: Vercel default function timeout is 10s.
// SSE evaluation streams need up to 60s for 4 parallel OpenAI calls.
export const maxDuration = 60;

// In-memory rate limiter: max 1 concurrent evaluation per proposalId.
// NOTE (review concern): This Set does NOT survive Vercel cold starts or
// span across multiple serverless instances. Two concurrent requests to
// different instances can bypass this check. For production, use a
// distributed lock (Redis, Vercel KV, or on-chain status check).
// For v1, this prevents the most common case (user double-clicking).
const activeEvaluations = new Set<string>();

const requestBodySchema = z.object({
  proposalId: z.string().min(1),
  // Addresses review concern: unbounded proposalText could generate
  // very large OpenAI requests. 10000 chars is ~2500 tokens, well
  // within GPT-4o's context window while preventing abuse.
  proposalText: z.string().min(1).max(10000),
});
```

**`POST` handler:**

1. Parse and validate request body:
   ```typescript
   const parseResult = requestBodySchema.safeParse(await request.json());
   if (!parseResult.success) {
     return Response.json(
       { error: "Invalid request", details: parseResult.error.flatten().fieldErrors },
       { status: 400 },
     );
   }
   const { proposalId, proposalText } = parseResult.data;
   ```

2. Rate limit check:
   ```typescript
   if (activeEvaluations.has(proposalId)) {
     return Response.json(
       { error: "Evaluation already in progress for this proposal" },
       { status: 429 },
     );
   }
   ```

3. Create SSE stream with try/finally cleanup (addresses review concern: activeEvaluations cleanup on failure):
   ```typescript
   const stream = new ReadableStream({
     async start(controller) {
       const encoder = new TextEncoder();
       const sendEvent = (event: EvaluationProgressEvent) => {
         controller.enqueue(encoder.encode(`data: ${JSON.stringify(event)}\n\n`));
       };

       activeEvaluations.add(proposalId);
       try {
         await orchestrateEvaluation(proposalId, proposalText, sendEvent);
         controller.close();
       } catch (error) {
         sendEvent({
           type: "failed",
           error: error instanceof Error ? error.message : "Unknown error",
         });
         controller.close();
       } finally {
         // Addresses review concern: always remove from activeEvaluations,
         // even if orchestration throws or stream is aborted.
         activeEvaluations.delete(proposalId);
       }
     },
   });
   ```

4. Return SSE response:
   ```typescript
   return new Response(stream, {
     headers: {
       "Content-Type": "text/event-stream",
       "Cache-Control": "no-cache",
       "Connection": "keep-alive",
     },
   });
   ```
