**src/lib/evaluation/use-evaluation.ts** — Custom React hook that consumes SSE from POST /api/evaluate.

```typescript
"use client";
import { useState, useCallback } from "react";
import type { EvaluationProgressEvent } from "./orchestrator";
import type { EvaluationOutput, EvaluationDimension, ProposalEvaluation } from "./schemas";
```

`useEvaluation()` returns:
```typescript
{
  startEvaluation: (proposalId: string, proposalText: string) => Promise<void>;
  status: "idle" | "evaluating" | "evaluated" | "failed";
  completedDimensions: Map<EvaluationDimension, EvaluationOutput>;
  failedDimensions: Set<EvaluationDimension>;
  aggregate: { weightedScore: number; completedDimensions: number } | null;
  evaluation: ProposalEvaluation | null;
  ipfsCid: string | null;
  txHash: string | null;
  error: string | null;
  isLoading: boolean;
}
```

`startEvaluation` implementation:
1. Set status to "evaluating", clear previous state.
2. `const response = await fetch("/api/evaluate", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ proposalId, proposalText }) });`
3. If response not ok (400/429), parse error JSON, set error state, return.
4. Get reader: `const reader = response.body?.getReader();` and `const decoder = new TextDecoder();`
5. Read loop: `while (true) { const { done, value } = await reader.read(); if (done) break; ... }`
6. Parse SSE lines: split decoded text by `\n\n`, filter lines starting with `data: `. For each data line, `JSON.parse` the payload inside a try/catch block, then validate the result using Zod `safeParse` with a discriminated union schema matching all 7 event types (started, dimension_complete, dimension_failed, aggregate_computed, stored, complete, failed). Define `evaluationProgressEventSchema` as `z.discriminatedUnion("type", [...])` at module scope (or import from orchestrator if exported). On safeParse failure, log a warning and skip the event — never use `as EvaluationProgressEvent` or any `as Type` cast.
7. Handle each event type:
   - `started`: no-op (already set status)
   - `dimension_complete`: add to `completedDimensions` Map
   - `dimension_failed`: add to `failedDimensions` Set
   - `aggregate_computed`: set aggregate state
   - `stored`: set ipfsCid and txHash
   - `complete`: set evaluation and status to "evaluated"
   - `failed`: set error and status to "failed"

Handle SSE line buffering: accumulate partial chunks in a buffer, split on `\n\n`, process complete events, keep remainder for next chunk.
