# Phase 2 — UI Design Contract

> Visual and interaction contract for the AI Evaluation Pipeline phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** UI-01 (evaluation results page), UI-03 (live judging progress), EVAL-07 (real-time progress), EVAL-08 (before/after prompt comparison). Phase 2 introduces the primary evaluation experience: watching judges evaluate in real-time, then viewing the full results breakdown.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, progress indicator gaps |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, gap between dimension cards |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Major section breaks (e.g., progress section to results section) |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Touch targets: minimum 44x44px on "Start Evaluation" button and any interactive elements
- Progress card minimum height: 80px (ensures agent status row is comfortably readable)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` | Page background, evaluation container background |
| Secondary (30%) | `var(--card)` | Dimension cards, progress panel, prompt comparison panels |
| Accent (10%) | `var(--primary)` | "Start Evaluation" button, active progress indicator, aggregate score numeral |
| Destructive | `var(--destructive)` | Not used in Phase 2 (no destructive actions) |

Accent reserved for: "Start Evaluation" CTA button background, currently-active progress step indicator, aggregate score numeral highlight, and recommendation badge for "strong_approve".

### Semantic Score Colors

Score bands use muted semantic colors for recommendation badges only. These are NOT accent colors -- they are contextual indicators within dimension cards.

| Recommendation | Badge Color | Badge Background |
|---------------|-------------|------------------|
| strong_approve | `var(--primary-foreground)` | `var(--primary)` |
| approve | `var(--foreground)` | `var(--secondary)` |
| needs_revision | `var(--warning-foreground, var(--foreground))` | `var(--warning, hsl(38 92% 50%))` |
| reject | `var(--destructive-foreground)` | `var(--destructive)` |

> Warning color may not exist in all shadcn presets. Fallback: use `hsl(38 92% 50%)` for amber/warning. If the preset includes a warning token, use it.

### Progress State Colors

| State | Indicator Color |
|-------|----------------|
| Pending (not started) | `var(--muted)` |
| Running (in progress) | `var(--primary)` with pulse animation |
| Complete | `var(--primary)` solid |
| Failed | `var(--destructive)` |

| Breakpoint | Layout Behavior |
|------------|----------------|
| Default (mobile-first) | Single column, stacked vertically, full-width cards |
| `sm` (640px) | Dimension cards remain single column but with more horizontal padding |

### Live Judging Progress Layout

During evaluation (status: "evaluating"), the progress panel is the primary visual element:

```
+-------------------------------------------+
| Evaluating Proposal...                    |
+-------------------------------------------+
| [*] Technical Feasibility (25%)    Done   |
| [~] Impact Potential (30%)     Running... |
| [ ] Cost Efficiency (20%)       Pending   |
| [ ] Team Capability (25%)       Pending   |
+-------------------------------------------+
| 1 of 4 judges complete                   |
+-------------------------------------------+

Legend:
[*] = solid circle, var(--primary)
[~] = pulsing circle, var(--primary) with animate-pulse
[ ] = empty circle, var(--muted)
```

## Interaction Contracts

### Start Evaluation

| Interaction | Behavior |
|-------------|----------|
| Click "Start Evaluation" | Button transitions to disabled state with spinner. Text changes to "Starting..." Mutation fires to Convex, which starts the durable workflow. |
| Evaluation already in progress | Button hidden. Progress panel visible instead. |
| Evaluation already complete | Button hidden. Results displayed. |

### Real-Time Progress (UI-03, EVAL-07)

| Interaction | Behavior |
|-------------|----------|
| Agent completes | Convex subscription fires. That agent's row transitions from "Running" to "Done" with score preview. Next pending agent transitions to "Running" (visual only -- agents actually run in parallel, but results arrive one at a time). |
| All agents complete | Progress panel collapses to compact summary. Dimension cards animate in (fade-in, 200ms, respect `prefers-reduced-motion`). Aggregate score appears. |
| Agent fails | That agent's row shows "Failed" with `var(--destructive)` color. Remaining agents continue. |

### Prompt Comparison (EVAL-08)

| Interaction | Behavior |
|-------------|----------|
| Mobile tab switch | Tabs component switches between "Naive" and "Structured" panels. No network request (static content). |
| Desktop view | Side-by-side panels, each showing prompt input and output. Left: naive. Right: structured. |
| Scroll within panel | Each panel scrolls independently (overflow-y-auto, max-h-[400px]). |

### Dimension Card Expansion

| Interaction | Behavior |
|-------------|----------|
| Default state | Card shows score, recommendation badge, and first 2 lines of justification (truncated with ellipsis). |
| Click/tap card | Card expands to show full justification and all key findings. No page navigation. |
| Click/tap again | Card collapses back to truncated state. |

## State Contracts

### Loading States

| State | Visual |
|-------|--------|
| Page initial load | Skeleton for aggregate score area + 4 skeleton dimension cards in grid layout |
| Evaluation not started | Aggregate score area shows "Start Evaluation" button. Dimension card area shows empty state. |
| Evaluation in progress | Progress panel active. Completed dimension cards render as they arrive. Pending slots show skeleton cards. |
| Evaluation complete | All dimension cards rendered. Aggregate score visible. Progress panel collapsed to "4/4 judges complete". |

### Empty States

| State | Visual |
|-------|--------|
| Proposal not yet evaluated | Card with `Brain` Lucide icon (48px, `var(--muted-foreground)`). Heading: "Ready for evaluation". Body: "Click Start Evaluation to have four AI judges assess this proposal across Technical Feasibility, Impact, Cost Efficiency, and Team Capability." CTA: "Start Evaluation" button. |
| No proposals exist (list page) | Handled by Phase 1 UI-SPEC (not this phase). |

### Error States

| State | Visual |
|-------|--------|
| Evaluation workflow failed entirely | Card with `AlertTriangle` Lucide icon (48px, `var(--destructive)`). Heading: "Evaluation failed". Body: "The evaluation could not be completed. This may be due to a temporary issue with the AI service. Try again in a few moments." CTA: "Retry Evaluation" button. |
| Individual agent failed | Progress row shows "Failed" in `var(--destructive)`. Dimension card slot shows: "This dimension could not be evaluated." Aggregate score computed from available dimensions with asterisk: "73/100*" and footnote "* Based on {N} of 4 dimensions". |
| Network error during progress | Toast notification: "Connection lost. Reconnecting..." Convex subscription auto-reconnects. No user action needed. |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Progress live region | Progress panel uses `aria-live="polite"` so screen readers announce agent completions |
| Score announcement | Aggregate score uses `aria-label="Overall evaluation score: {score} out of 100, rated {band}"` |
| Recommendation badge | Badge includes `aria-label="Recommendation: {label}"` for screen reader clarity |
| Expandable cards | DimensionCard uses `aria-expanded` attribute. Toggle button has `aria-label="Show full evaluation for {dimension}"` |
| Touch targets | Minimum 44x44px for "Start Evaluation" button, card expansion toggle, tab switches |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 for body, 3:1 for large text). Badge colors tested against their backgrounds. |
| Motion | Progress pulse animation respects `prefers-reduced-motion`. Dimension card fade-in disabled when motion reduced. |
| Keyboard navigation | Tab key moves through progress panel, dimension cards (expanded/collapsed), prompt comparison tabs. Enter/Space toggles card expansion. |
