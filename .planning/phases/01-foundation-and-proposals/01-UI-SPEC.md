# Phase 1 — UI Design Contract

> Visual and interaction contract for the proposal submission, listing, and detail pages. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | default (initialize during execution with `bunx shadcn@latest init`) |
| Component library | radix (via shadcn/ui) |
| Icon library | lucide-react (ships with shadcn/ui) |
| Font | Inter (via `next/font/google`) |

**shadcn components needed for Phase 1:**
`button`, `card`, `input`, `textarea`, `badge`, `separator`, `skeleton`, `label`, `form` (optional — can use native form + Zod)

**Note:** Project is greenfield — no `package.json` exists yet. shadcn init happens during plan execution after Next.js bootstrapping.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, form field internal padding |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding within pages, card content padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page vertical padding (top/bottom of content area) |
| 3xl | 64px | Hero/header vertical spacing |

Exceptions: none

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Proposal descriptions, general text |
| Label | 14px | 400 (regular) | 1.4 | Form field labels, badge text, metadata captions, table headers |
| Heading | 24px | 600 (semibold) | 1.2 | Page titles ("Submit a Proposal", "All Proposals"), card titles |
| Display | 32px | 600 (semibold) | 1.1 | Landing header only ("IPE City Grants") |

**Font stack:** `Inter, ui-sans-serif, system-ui, -apple-system, sans-serif`

**Declared weights:** 2 total — 400 (regular) and 600 (semibold). Labels are differentiated from Body by size (14px vs 16px), not weight. Display uses 600 (same as Heading) and is differentiated by size (32px vs 24px).

## Color

The palette uses shadcn/ui default neutral theme with a single deliberate accent. The aesthetic is minimal, technical, and trustworthy — not playful.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` / white | Page background, form backgrounds |
| Secondary (30%) | `hsl(0 0% 98%)` / near-white gray | Card backgrounds, proposal list items, nav bar background, sidebar |
| Accent (10%) | `hsl(221 83% 53%)` / blue-600 | Primary CTA buttons, active status badges, link hover underlines, proposal score highlights |
| Destructive | `hsl(0 84% 60%)` / red-500 | Delete/discard proposal button only |
| Muted text | `hsl(0 0% 45%)` / gray-500 | Secondary text, timestamps, metadata labels |
| Border | `hsl(0 0% 90%)` / gray-200 | Card borders, form field borders, separators |

**Accent reserved for:**
1. "Submit Proposal" primary CTA button
2. Status badge background when status is "evaluated" (success state)
3. Hyperlinks on hover
4. Score number display (when scores exist in later phases)

**Dark mode:** Not in Phase 1 scope. Use shadcn default light theme only.

## Component Inventory

### Page: Proposal List (`/proposals`)

**Focal point:** Primary visual anchor is the "Submit Proposal" CTA button in the PageHeader, positioned top-right at the entry point of the grant submission flow.

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| ProposalCard | `Card` | Displays title, truncated description (120 chars + ellipsis), status badge, budget. Entire card is clickable (links to detail). |
| StatusBadge | `Badge` | Three variants: `submitted` (gray bg, gray text), `evaluating` (amber bg, amber text), `evaluated` (accent bg, white text) |
| EmptyState | Custom (no shadcn) | Centered illustration-free layout: heading + body + CTA button linking to `/proposals/new` |
| ProposalListSkeleton | `Skeleton` | 3 card-shaped skeletons while data loads from IPFS/chain |
| PageHeader | Custom | Heading + "Submit Proposal" CTA button aligned right |

**Layout:** Single column, max-width 768px, centered. Cards stack vertically with 16px gap.

### Page: Proposal Detail (`/proposals/[id]`)

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| ProposalHeader | Custom | Title (Heading size), status badge, submission date |
| ProposalContent | `Card` | Full description, team info, budget displayed as labeled sections with `Separator` between them |
| ExternalLinks | Custom | List of external links as styled anchor tags with external-link icon |
| IPFSHash | Custom | Monospace display of IPFS CID, truncated with copy-to-clipboard button |
| ChainReference | Custom | On-chain transaction hash, truncated with link to block explorer |
| BackLink | Custom | "Back to proposals" link at top of page |
| DetailSkeleton | `Skeleton` | Full-page skeleton layout while IPFS content loads |

**Layout:** Single column, max-width 768px, centered. Content sections stack with 24px gap.

### Page: Submit Proposal (`/proposals/new`)

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| ProposalForm | `Card` wrapping form | Card container with form fields inside |
| TitleInput | `Input` | Text input, max 200 chars, required |
| DescriptionTextarea | `Textarea` | Multi-line, min 50 chars, max 5000, required. Show character count. |
| TeamInfoTextarea | `Textarea` | Multi-line, min 10 chars, max 2000, required |
| BudgetInput | `Input` | Numeric, positive, max 1,000,000. Prefix with "$" visual indicator. |
| ExternalLinksInput | Custom | Dynamic list — add/remove URL fields. Max 5 links. Each validates as URL. |
| SubmitButton | `Button` | Primary accent button. Shows loading spinner during submission. Disabled when form invalid. |
| FormError | Custom | Red text below field with validation message from Zod |
| SuccessRedirect | N/A | On success, redirect to `/proposals/[id]` with the new proposal |

**Layout:** Single column, max-width 640px, centered. Form fields stack with 16px gap. Labels above fields with 4px gap.

### Shared: App Shell

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| NavBar | Custom | Fixed top bar. Left: "IPE City Grants" text logo (links to `/proposals`). Right: "Submit Proposal" button (links to `/proposals/new`). Height: 64px. |
| Footer | Custom | Minimal: "Built for IPE City" + link to ipe.city. Height: 48px. Muted text. |
| MainContent | Custom | `<main>` with horizontal padding 16px (mobile) / 0px (desktop, content max-width handles centering). Vertical padding 48px top, 48px bottom. |

**Responsive breakpoints:**
- Mobile: < 640px — full-width cards with 16px horizontal padding
- Desktop: >= 640px — max-width containers (768px list/detail, 640px form), centered

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigation | All interactive elements reachable via Tab. Cards focusable as links. Form submittable via Enter. |
| Screen reader labels | Form fields have associated `<label>` elements. Status badges have descriptive text (not color-only). IPFS hash has "Copy to clipboard" aria-label on button. |
| Color contrast | Body text on white: minimum 4.5:1 (gray-500 on white = ~5.8:1). Accent on white buttons: white text on blue-600 = ~4.6:1. |
| Focus indicators | Visible focus ring on all interactive elements (2px accent outline, 2px offset) |
| Reduced motion | Skeleton pulse and hover transitions respect `prefers-reduced-motion: reduce` |
| Semantic HTML | `<nav>`, `<main>`, `<article>` (proposal cards), `<form>`, `<h1>`/`<h2>` hierarchy |
