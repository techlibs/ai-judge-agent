    Create two Solidity contracts following the ERC-8004 specification scope from PROJECT.md.

    **contracts/src/IdentityRegistry.sol:**
    - SPDX-License-Identifier: MIT, pragma solidity ^0.8.24
    - Extends OpenZeppelin ERC721 (`@openzeppelin/contracts/token/ERC721/ERC721.sol`)
    - Stores: mapping(uint256 => string) for agentURI (IPFS CID for project metadata)
    - `register(address owner, string calldata agentURI) external returns (uint256 tokenId)` — mints new ERC-721 token, stores agentURI, emits `ProjectRegistered(uint256 indexed tokenId, address indexed owner, string agentURI)`
    - `setAgentURI(uint256 tokenId, string calldata agentURI) external` — only token owner can update, emits `AgentURIUpdated(uint256 indexed tokenId, string agentURI)`
    - `getMetadata(uint256 tokenId) external view returns (address owner, string memory agentURI)` — returns owner + agentURI
    - Auto-incrementing tokenId counter starting at 1
    - `tokenURI` override returning agentURI
    - No access control beyond ERC-721 ownership (public registration for v1)

    **contracts/src/ReputationRegistry.sol:**
    - SPDX-License-Identifier: MIT, pragma solidity ^0.8.24
    - Stores: struct Feedback { address evaluator; uint256 score; string contentHash; uint256 timestamp; }
    - Stores: mapping(uint256 => Feedback[]) for tokenId => feedback array
    - `giveFeedback(uint256 tokenId, uint256 score, string calldata contentHash) external` — score must be 0-100, pushes Feedback, emits `FeedbackGiven(uint256 indexed tokenId, address indexed evaluator, uint256 score, string contentHash)`
    - `getSummary(uint256 tokenId) external view returns (uint256 feedbackCount, uint256 averageScore)` — returns count and average of all feedback scores
    - `readFeedback(uint256 tokenId, uint256 index) external view returns (address evaluator, uint256 score, string memory contentHash, uint256 timestamp)` — returns single feedback entry
    - `getFeedbackCount(uint256 tokenId) external view returns (uint256)` — returns feedback array length
    - Custom errors: `InvalidScore(uint256 score)`, `FeedbackIndexOutOfBounds(uint256 index, uint256 length)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External caller -> IdentityRegistry | Anyone can call register() — public registration for v1 |
| External caller -> ReputationRegistry | Anyone can call giveFeedback() — open feedback for v1 |
| Token owner -> setAgentURI | Only owner can modify their project URI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | ReputationRegistry.giveFeedback | mitigate | Validate score 0-100 with custom error `InvalidScore`; evaluator address stored immutably per feedback |
| T-01-02 | Tampering | IdentityRegistry.setAgentURI | mitigate | Require `msg.sender == ownerOf(tokenId)` check via ERC-721 ownership |
| T-01-03 | Denial of Service | IdentityRegistry.register | accept | Public registration means spam is possible; mitigated by gas costs on testnet. Rate limiting deferred to v2 |
| T-01-04 | Spoofing | ReputationRegistry.giveFeedback | accept | Anyone can submit feedback in v1; evaluator identity recorded. Access control deferred until evaluator registration exists |
| T-01-05 | Information Disclosure | Both contracts | accept | All data is public by design — evaluations are transparency-first |
</threat_model>
