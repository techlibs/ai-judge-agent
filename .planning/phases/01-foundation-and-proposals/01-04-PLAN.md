    **src/app/api/proposals/route.ts (GET — list all proposals):**
    Read all proposals from on-chain events. This is the "rebuildable from chain events" approach (STORE-04):

    1. Use viem `publicClient.getLogs()` to fetch all `ProjectRegistered` events from the IdentityRegistry contract
    2. For each event, extract `tokenId`, `owner`, `agentURI` (IPFS CID)
    3. For each tokenId, call `ReputationRegistry.getSummary(tokenId)` to get `feedbackCount` and `averageScore`
    4. Determine status: if feedbackCount > 0 then "evaluated", else "submitted" (evaluating is set during Phase 2 evaluation flow)
    5. Return JSON array:
       ```typescript
       interface ProposalListItem {
         tokenId: string;
         owner: string;
         ipfsCID: string;
         status: "submitted" | "evaluating" | "evaluated";
         feedbackCount: number;
         averageScore: number;
         // Minimal IPFS data fetched for list display (title, description excerpt, budget)
         title: string;
         description: string; // full — truncation is client-side
         budget: number;
         submittedAt: string;
       }
       ```
    6. For list view, fetch IPFS content for each proposal to get title/description/budget. Use `Promise.all()` for parallel fetches. Create a `proposalContentSchema` (Zod) to validate IPFS responses, and pass it to `fetchFromIPFS`.
    7. Sort by submittedAt descending (newest first).
    8. Error handling: if IPFS fetch fails for a proposal, include it with title "Content unavailable" and empty description.

    ```typescript
    import { NextResponse } from "next/server";
    import { z } from "zod";
    import { getPublicClient } from "@/lib/chain/client";
    import {
      IDENTITY_REGISTRY_ABI,
      REPUTATION_REGISTRY_ABI,
      getContractAddresses,
    } from "@/lib/chain/contracts";
    import { fetchFromIPFS } from "@/lib/ipfs/client";

    const proposalContentSchema = z.object({
      title: z.string(),
      description: z.string(),
      teamInfo: z.string(),
      budget: z.number(),
      externalLinks: z.array(z.string()),
      submittedAt: z.string(),
    });

    export async function GET() {
      try {
        const publicClient = getPublicClient();
        const { identityRegistry, reputationRegistry } = getContractAddresses();

        // Fetch all ProjectRegistered events
        const logs = await publicClient.getLogs({
          address: identityRegistry,
          event: {
            type: "event",
            name: "ProjectRegistered",
            inputs: [
              { name: "tokenId", type: "uint256", indexed: true },
              { name: "owner", type: "address", indexed: true },
              { name: "agentURI", type: "string", indexed: false },
            ],
          },
          fromBlock: 0n,
          toBlock: "latest",
        });

        // Enrich each proposal with reputation data and IPFS content
        const proposals = await Promise.all(
          logs.map(async (log) => {
            const tokenId = log.args.tokenId?.toString() ?? "0";
            const owner = log.args.owner ?? "0x0";
            const ipfsCID = log.args.agentURI ?? "";

            // Get reputation summary
            const [feedbackCount, averageScore] = await publicClient.readContract({
              address: reputationRegistry,
              abi: REPUTATION_REGISTRY_ABI,
              functionName: "getSummary",
              args: [BigInt(tokenId)],
            });

            // Determine status from feedback count
            const count = Number(feedbackCount);
            const status = count > 0 ? "evaluated" : "submitted";

            // Fetch IPFS content for display
            let title = "Content unavailable";
            let description = "";
            let budget = 0;
            let submittedAt = "";
            try {
              const content = await fetchFromIPFS(ipfsCID, proposalContentSchema);
              title = content.title;
              description = content.description;
              budget = content.budget;
              submittedAt = content.submittedAt;
            } catch {
              // IPFS gateway may be temporarily unavailable
            }

            return {
              tokenId,
              owner,
              ipfsCID,
              status,
              feedbackCount: count,
              averageScore: Number(averageScore),
              title,
              description,
              budget,
              submittedAt,
            };
          }),
        );

        // Sort newest first
        proposals.sort((a, b) => b.submittedAt.localeCompare(a.submittedAt));

        return NextResponse.json(proposals);
      } catch {
        return NextResponse.json(
          { error: "Failed to fetch proposals" },
          { status: 500 },
        );
      }
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL params -> API route | tokenId from URL could be any string |
| IPFS gateway -> client | Content fetched from external gateway could be tampered |
| On-chain events -> API | Event data is trustworthy (immutable on chain) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | /api/proposals/[tokenId] | mitigate | tokenId is validated by on-chain contract call — invalid IDs cause contract revert, caught as 404. No raw SQL or DB queries to inject. |
| T-04-02 | Information Disclosure | /api/proposals error responses | mitigate | Return generic "Failed to fetch proposals" / "Proposal not found", never expose RPC URLs or internal error details |
| T-04-03 | Tampering | IPFS content display | mitigate | Content fetched from IPFS is validated with Zod schema before use and rendered as text via React (auto-escaped). No dangerouslySetInnerHTML. External links open in new tab with rel="noopener noreferrer". |
| T-04-04 | Spoofing | External links in proposals | accept | Links are user-submitted URLs displayed as-is. Users should verify links before clicking. No server-side link validation beyond URL format. |
</threat_model>
