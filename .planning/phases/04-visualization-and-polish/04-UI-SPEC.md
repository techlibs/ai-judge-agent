# Phase 4 — UI Design Contract

> Visual and interaction contract for the Visualization and Polish phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> 
> **Scope:** UI-02 (score visualization chart) + UI-05 (responsive design audit). No backend changes. Consumes evaluation data produced by Phase 2.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Preset | Inherited from Phase 1 initialization |
| Component library | Radix (via shadcn/ui) |
| Icon library | Lucide React (shadcn default) |
| Font | Inherited from Phase 1 (recommend Inter or Geist) |

> Phase 4 does not initialize the design system. It inherits whatever Phase 1 establishes and adds the `chart` component.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, chart label padding |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, gap between chart and details |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Chart minimum height: 250px (required by Recharts ResponsiveContainer for initial render measurement)
- Touch targets: minimum 44x44px on all interactive elements (mobile accessibility)

| Role | Size | Weight | Line Height |

**Font sizes: 4 (14px, 16px, 20px, 28px). Font weights: 2 (400, 600).**

### Phase 4 Specific Typography

| Element | Size | Weight | Notes |
|---------|------|--------|-------|
| Chart axis labels (all viewports) | 14px | 400 | Use abbreviated labels on mobile: "Tech", "Impact", "Cost", "Team". Full labels on desktop: "Technical", "Impact", "Cost", "Team" |
| Aggregate score display | 28px | 600 (semibold) | Large numeral showing weighted score (0-100). Use `var(--chart-1)` accent color for visual prominence. |
| Score value in tooltip | 16px | 600 | Score number inside chart tooltip |
| Dimension weight label | 14px | 400 | e.g. "25%" shown alongside dimension name |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` | Page background, chart container background |
| Secondary (30%) | `var(--card)` | Score summary card, dimension detail cards, chart card wrapper |
| Accent (10%) | `var(--chart-1)` | Radar polygon fill + stroke, aggregate score numeral |
| Destructive | `var(--destructive)` | Not used in Phase 4 (no destructive actions) |

Accent reserved for: radar chart polygon fill (at 0.5 opacity), radar chart polygon stroke, aggregate score numeral, and primary CTA button background.

### Chart-Specific Colors

| Token | Purpose | Value |
|-------|---------|-------|
| `--chart-1` | Radar polygon fill + stroke | shadcn default (blue/indigo family) |
| `--chart-2` | Reserved for comparison overlay (future) | shadcn default |
| Radar grid lines | Polar grid | `var(--border)` at default opacity |
| Axis text | PolarAngleAxis labels | `var(--muted-foreground)` |

> All chart colors use shadcn CSS variable tokens. No hardcoded hex values. Dark mode support is automatic via CSS variable switching.

### Breakpoint Strategy

| Breakpoint | Layout Behavior |
|------------|----------------|
| Default (mobile-first) | Single column, stacked vertically, full-width elements |
| `sm` (640px) | Dimension cards switch to 2-column grid |
| `md` (768px) | Chart + aggregate score go side-by-side; main content 2-column |
| `lg` (1024px) | Dimension cards switch to 4-column grid (one per dimension) |
| `xl` (1280px) | Content constrained to `max-w-6xl` with centered margin |

## Interaction Contracts

### Radar Chart

| Interaction | Behavior |
|-------------|----------|
| Hover on radar polygon vertex | Tooltip appears showing dimension name, score value (0-100), and weight percentage |
| Tap on vertex (mobile) | Same tooltip behavior as hover |
| Chart resize | Automatic via Recharts ResponsiveContainer inside ChartContainer. No manual handling. |
| Incomplete evaluation (< 4 dimensions) | Show Skeleton placeholder instead of chart. Do not render partial radar. |
| All scores are 0 | Render radar with collapsed polygon (dot at center). Show "All dimensions scored 0" caption. |

### Responsive Navigation

| Interaction | Behavior |
|-------------|----------|
| Mobile nav toggle | If nav exists, collapse behind hamburger icon (Lucide `Menu` icon, 24px). Tap to expand/collapse. |
| Scroll on mobile | Sticky header with nav if applicable. Content scrolls beneath. |

## State Contracts

### Loading States

| State | Visual |
|-------|--------|
| Evaluation in progress (< 4 agents complete) | Skeleton pulse in chart area (same `min-h-[250px]`). Text: "Evaluating... {N} of 4 judges complete" |
| Evaluation complete (all 4 agents done) | Radar chart renders with full polygon. Aggregate score numeral visible. |
| Page loading (initial) | Skeleton for entire ScoreSummaryCard area |

### Empty States

| State | Visual |
|-------|--------|
| Proposal not yet evaluated | Card with `FileBarChart` Lucide icon (48px, `var(--muted-foreground)`). Heading: "No evaluation yet". Body: "This proposal is awaiting evaluation. Results will appear here once all four judges complete their assessment." |

### Error States

| State | Visual |
|-------|--------|
| Evaluation partially failed | Show completed dimensions in chart (if 3+ complete). Banner: "Some evaluations could not be completed. Showing available results." Use `var(--warning)` or `var(--muted-foreground)` for banner background. |
| Chart render failure | Fallback to simple table showing dimension name + score numerically. No chart. Log error to console. |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Chart alternative | `aria-label` on ChartContainer: "Radar chart showing evaluation scores across four dimensions" |
| Score screen reader | Visually hidden table with dimension + score values alongside the radar chart |
| Touch targets | Minimum 44x44px for all tappable elements (buttons, links, nav items) |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 for body, 3:1 for large text). Ensured by shadcn defaults. |
| Motion | Respect `prefers-reduced-motion`. Disable chart entry animations when user prefers reduced motion. |
| Hamburger menu icon | `aria-label="Toggle navigation"` on the mobile nav toggle button (Lucide `Menu` icon, 24px). Icon-only interactive element must have accessible label. |
