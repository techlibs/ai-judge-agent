# Phase 3 — UI Design Contract

> Visual and interaction contract for the Reputation History and Querying phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** CHAIN-05 (reputation history per project queryable from on-chain events). This phase adds reputation history views and on-chain verification links. It consumes on-chain publication data produced by Phase 2 (stored in Convex `onchainPublications` table with `txHash`, `blockNumber`, `evaluationHash`).

## Interaction Contracts

### Transaction Hash

| Interaction | Behavior |
|-------------|----------|
| View tx hash | Displayed truncated: first 6 + last 4 characters (e.g. `0xab12...ef34`). Full hash visible on hover via Tooltip. |
| Copy tx hash | Click copy icon (Lucide `Copy`, 16px) next to hash. On click: copy full hash to clipboard, icon changes to Lucide `Check` for 2 seconds, then reverts. |
| View on chain | Click "View on chain" link (Lucide `ExternalLink`, 16px). Opens `https://sepolia.basescan.org/tx/{txHash}` in new tab (`target="_blank" rel="noopener noreferrer"`). |

### Reputation Summary

| Interaction | Behavior |
|-------------|----------|
| Agent ID display | Show the ERC-8004 agent ID as "#42" format. On click or hover, show Tooltip with full details: "ERC-8004 Identity Token #42 on Base Sepolia". |
| Average score | Non-interactive display. Shows weighted average from all evaluations. |

### History List

| Interaction | Behavior |
|-------------|----------|
| Click evaluation row | Navigate to the evaluation detail page for that specific evaluation (existing Phase 2 page). |
| Loading more entries | If more than 20 evaluations exist (unlikely in v1), paginate with "Load more" button at bottom. Not infinite scroll. |

## State Contracts

### Loading States

| State | Visual |
|-------|--------|
| History loading (initial) | Skeleton pulse for ReputationSummaryCard (1 card). Below: 3 skeleton rows matching history entry dimensions. |
| On-chain data refreshing | Existing content remains visible. No spinner overlay. Subtle `opacity-50` on the refreshing section if needed. |

### Empty States

| State | Visual |
|-------|--------|
| Project has no evaluations yet | Card with Lucide `History` icon (48px, `var(--muted-foreground)`). Heading: "No reputation history". Body: "This project has not been evaluated yet. Reputation data will appear here after the first evaluation is published on-chain." |
| Project not registered on-chain | Card with Lucide `LinkOff` icon (48px, `var(--muted-foreground)`). Heading: "Not yet on-chain". Body: "This project has not been registered on the blockchain yet. On-chain identity is created when a proposal is first submitted." |

### Error States

| State | Visual |
|-------|--------|
| Failed to load on-chain data | Card with Lucide `AlertTriangle` icon (48px, `var(--muted-foreground)`). Heading: "Could not load reputation data". Body: "We were unable to read on-chain reputation data. This may be a temporary network issue. Please try again later." Retry button: "Try again" (secondary variant). |
| Individual tx confirmation failed | Badge shows "Failed" in `var(--destructive)` color. Row still visible with score and timestamp from Convex data. Tooltip: "This evaluation could not be confirmed on-chain." |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| History table semantics | Use `<table>` on desktop (via shadcn Table component). On mobile card layout, use `role="list"` with `role="listitem"` on each entry. |
| External links | All "View on chain" links include `aria-label="View transaction {truncatedHash} on Base Sepolia block explorer"` and open in new tab with visual indicator (ExternalLink icon). |
| Copy button | `aria-label="Copy transaction hash to clipboard"`. After copy: `aria-label="Transaction hash copied"`. |
| Tooltip keyboard access | All Tooltip triggers are focusable. Tooltip appears on focus as well as hover. |
| Score screen reader | Each history entry score has `aria-label="Score: {score} out of 100"`. |
| Status badge | Badge includes `aria-label="On-chain status: {status}"` for screen readers. |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 for body, 3:1 for large text). Ensured by shadcn defaults. |
