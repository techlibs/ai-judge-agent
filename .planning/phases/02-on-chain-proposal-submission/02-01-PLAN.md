```solidity
function submitProposal(
    string calldata title,
    bytes32 contentCid,
    bytes32 repoUrlHash,
    bytes32 demoUrlHash,
    ProposalDomain domain
) external returns (bytes32 proposalId) {
    if (contentCid == bytes32(0)) revert InvalidContentCid();
    proposalId = keccak256(abi.encodePacked(msg.sender, title, block.timestamp));
    if (proposals[proposalId].submittedAt != 0) revert ProposalAlreadyExists(proposalId);
    proposals[proposalId] = Proposal({
        submitter: msg.sender,
        submittedAt: uint48(block.timestamp),
        status: ProposalStatus.Submitted,
        contentCid: contentCid,
        repoUrlHash: repoUrlHash,
        demoUrlHash: demoUrlHash,
        domain: domain
    });
    proposalIds.push(proposalId);
    emit ProposalSubmitted(proposalId, msg.sender, contentCid, domain, uint48(block.timestamp));
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Proposer -> ProposalRegistry | Untrusted external input: title string, contentCid bytes32, repoUrlHash, demoUrlHash, domain enum |
| GrantRouter -> ProposalRegistry | Trusted (onlyOwner): status transitions |
| ProposalRegistry -> TextStorage | Internal library call: backupProposalText passes proposer-supplied content bytes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | submitProposal | accept | msg.sender is the submitter identity -- no spoofing possible at contract level |
| T-02-02 | Tampering | Proposal struct | mitigate | D-09: proposals are immutable, no edit/delete functions. Struct fields set once in submitProposal, only status changes via onlyOwner setStatus |
| T-02-03 | Denial of Service | submitProposal | accept | D-08: open submission allows spam. Gas cost provides natural rate limiting. Protocol-level rate limiting deferred per CONTEXT.md |
| T-02-04 | Elevation of Privilege | setStatus | mitigate | onlyOwner modifier on setStatus (D-10). Only GrantRouter (owner) can transition status. Validated in test_setStatus_onlyOwner |
| T-02-05 | Input Validation | submitProposal contentCid | mitigate | Revert on bytes32(0) contentCid via InvalidContentCid error. Prevents empty/null content hashes |
| T-02-06 | Information Disclosure | proposalId determinism | accept | proposalId = keccak256(sender, title, timestamp) is predictable. Front-running is low-risk for grant proposals (not financial). Accept per research pitfall #3 analysis |
| T-02-07 | Tampering | backupProposalText | accept | Anyone can call backupProposalText for any existing proposal with arbitrary content. The function only emits events -- does not modify state. Content verification is off-chain (compare event hash with contentCid). Low risk. |
</threat_model>
