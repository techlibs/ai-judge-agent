Both GSD and Ralph's judge prompts receive proposals that include GitHub and video links. ## Methodology notes

- **Determinism**: all scores are single-shot LLM calls with temperature 0 (per Mastra scorer defaults). Re-running may produce ±1-3 point jitter per dimension but the rank ordering is stable in our observations.
- **Model used**: `gpt-4o` across both GSD and Ralph (post-migration).
- **IPFS persistence**: GSD persists proposal content on IPFS and records hash + owner on-chain (deployer wallet `0xa7cEd6c599…0f742`, Base network). Ralph evaluates in memory and does not persist.
- **Cost per evaluation** (Ralph, approximate): 4 dimensions × ~1500 prompt tokens × $0.005/1k input + ~500 output tokens × $0.015/1k output = **~$0.06 per proposal** at gpt-4o rates. Not measured empirically.

**Judge discrimination validated.** Across both implementations (Ralph direct API + GSD UI) the system produces a clean bimodal split:

- **Real public-goods projects** (rotki, Hypercerts, Kiwi News): 71–83/100 aggregate, all `fund` or `strong_fund`, high confidence.
- **Adversarial fakes** (rug-pull, vaporware, impersonation): 9–35/100 aggregate, all `reject`, mostly low confidence (correctly — the inputs are literal nonsense).

Worst real (Kiwi News, 71) is **31 points above** best fake (Nethereum v2, 35). There is no overlap in the score distributions. The judges not only reject the fakes, they **cite the specific adversarial signals** in their written justifications — lambo line items, Satoshi impersonation, TBD tech stack, undisclosed algorithms — which is stronger evidence of discrimination than the numerical gap alone.

# Appendix — Spec Kit results (added 2026-04-20)

## Method

The Cloud Run Spec Kit service (`agent-reviewer-speckit`) gates its submission endpoint `/api/webhooks/proposals` behind an API key lookup against a seeded `platformIntegrations` row plus an optional HMAC `X-Signature-256`, and no active API key was seeded into the Cloud Run instance. Rather than mutate production seed data or redeploy to alter the key list, the pure judge pipeline was driven locally against the same worktree code (`.worktrees/speckit/src/evaluation/agents/runner.ts` + `sanitization.ts`) via the driver at `.worktrees/speckit/scripts/evidence-run.ts`. The driver:

1. Loads all 6 fixtures (3 real projects as structured inputs derived from `test-fixtures/real-grants-with-media/*.md`, 3 fakes verbatim from `test-fixtures/fake-grants/*.json`).
2. Runs `sanitizeProposal()` (PII redaction, URL stripping, team profile hashing) exactly as the production pipeline does.
3. Calls `runAllDimensions(sanitized, null)` — 4 parallel `generateObject` calls against `gpt-4o` using Spec Kit's `DimensionScoreSchema` and prompts verbatim from `src/evaluation/agents/prompts.ts` (`PROMPT_VERSION = 1.0.0`).
4. Computes the weighted final score with `computeWeightedScore` (tech 25 / impact 30 / cost 20 / team 25), reputation multiplier pinned to 1.0 (equivalent to reputationIndex 0 — no on-chain lookup).
5. Writes raw JSON per fixture to `docs/e2e-evidence/raw-json/speckit_*.json` plus `speckit_summary.json`.

OpenAI-only (`@ai-sdk/openai` + `ai` v4 from the worktree's own deps). Anthropic not wired.

## Enrichment status (GitHub + video tools)

**NOT wired into Spec Kit's judge pipeline.**

- `src/evaluation/agents/tools/extract-github.ts` and `extract-video.ts` exist and have unit tests.
- They are imported **only** by `src/evaluation/agents/proposal-assistant.ts` (the pre-submission conversational helper), not by `runner.ts` (the judge loop).
- `sanitization.ts` explicitly **strips all URLs** (`URL_PATTERN` → `[URL_REDACTED]`) before judges see the proposal, so even if a proposer supplies a GitHub or video link, the judges never see it.
- Net: on Spec Kit, the judges score on narrative text only — no repo star counts, no README excerpts, no video transcripts, no release cadence. This is a deliberate sanitization-driven design choice and would be a non-trivial change to re-wire (it would require threading enriched context into `SanitizedProposal` or adding a parallel enrichment field that bypasses the URL-stripper). Left as a documented gap.
