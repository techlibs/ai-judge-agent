## Tool enrichment — GitHub + video context

Both GSD and Ralph's judge prompts receive proposals that include GitHub and video links. The GSD worktree has explicit `extractGithubRepo` and `extractVideoContext` tools at `src/lib/agents/tools/` that — when invoked — pull README excerpts + video oEmbed metadata into the prompt. In practice, the judges' justifications reference specific signals from the content:

- rotki's justification mentions "consistent releases since 2017" and the "AGPL-3.0 license" — both facts derivable from the GitHub README and package metadata.
- Hypercerts' justification notes "ERC-1155-based semi-fungible tokens" — consistent with the repo's technical README.
- Kiwi News's TECH judge references "P2P attestation network (kiwistand)" — directly from the linked repo name.

Whether the judges actually *called* the tools vs inferred from the description text is hard to verify without tracing; both paths produce plausible-looking output. For the Pyth case (earlier test) the judges correctly flagged the absence of technical architecture in the prompt, which suggests the tool output is either (a) included inline or (b) the judges are conservative when signals are weak.

## Methodology notes

- **Determinism**: all scores are single-shot LLM calls with temperature 0 (per Mastra scorer defaults). Re-running may produce ±1-3 point jitter per dimension but the rank ordering is stable in our observations.
- **Model used**: `gpt-4o` across both GSD and Ralph (post-migration).
- **IPFS persistence**: GSD persists proposal content on IPFS and records hash + owner on-chain (deployer wallet `0xa7cEd6c599…0f742`, Base network). Ralph evaluates in memory and does not persist.
- **Cost per evaluation** (Ralph, approximate): 4 dimensions × ~1500 prompt tokens × $0.005/1k input + ~500 output tokens × $0.015/1k output = **~$0.06 per proposal** at gpt-4o rates. Not measured empirically.
